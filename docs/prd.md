숫자 8자리는 컴퓨터 입장에서 **"복잡한 암호"가 아니라 "눈 깜빡할 사이의 간식"** 정도입니다.

20년 경력의 개발자이시니 연산량으로 계산해 보시면 바로 감이 오실 겁니다.

---

### 1. 왜 8자리 숫자는 금방 뚫릴까? (연산의 법칙)

숫자 8자리의 경우의 수는 아주 단순합니다.
* $00000000$부터 $99999999$까지 총 **1억 개**($10^8$)입니다.

이제 이걸 최신 GPU(예: RTX 4090) 한 장으로 무차별 대입(Brute-force) 한다고 가정해 보겠습니다.
* 보통 PDF 암호 해시를 대입하는 속도는 알고리즘에 따라 다르지만, 초당 수백만 번에서 수억 번까지 가능합니다.
* **초당 1억 번**을 대입할 수 있다면? **단 1초** 만에 전수 조사가 끝납니다.
* 조금 느린 알고리즘이라 **초당 100만 번**을 대입한다 해도, **약 100초(1분 40초)**면 8자리 숫자는 정복됩니다.



### 2. "복잡한 암호"의 기준이 바뀌었습니다

보통 보안 기사 시험이나 일반적인 가이드에서 말하는 "복잡한 암호"는 **조합의 가짓수**가 기하급수적으로 늘어나는 경우를 말합니다.

| 암호 구성 (8자리 기준) | 경우의 수 | 체감 난이도 |
| :--- | :--- | :--- |
| **숫자만 (0-9)** | $10^8$ (1억) | **1초 컷** |
| **영문 소문자만 (a-z)** | $26^8$ (약 2000억) | **수 분 내외** |
| **영문 대소문자 + 숫자** | $62^8$ (약 218조) | **수일 ~ 수개월** |
| **대소문자 + 숫자 + 특수문자** | $95^8$ (약 6.6경) | **수십 년 이상** |

### 3. PDF 보안의 특수성 (Key Stretching)

물론 현대의 PDF(AES-256)는 **Key Stretching**이라는 기술을 씁니다. 암호를 한 번 대입할 때마다 내부적으로 해시 연산을 수만 번 반복하게 만들어 컴퓨터가 피곤하게 만드는 거죠. 

하지만 아무리 연산을 느리게 꼬아놔도 **숫자 8자리는 경우의 수 자체가 너무 적습니다.** GPU 여러 대를 병렬로 붙인 해킹 장비(Password Cracker) 앞에서는 무의미한 방어입니다.

### 💡 결론: 선배님을 위한 요약

1.  **숫자 8자리:** 보안 전문가 입장에서는 **"암호가 없는 것이나 마찬가지"**인 수준입니다.
2.  **안전한 암호:** 최소한 영문 대소문자와 숫자를 섞은 **12자리 이상**은 되어야 현대의 연산 속도로부터 안전하다고 봅니다.
3.  **현실:** 만약 잊어버린 PDF 암호가 숫자 8자리라는 확신만 있다면, 파이썬으로 `itertools` 라이브러리를 써서 $0$부터 $99,999,999$까지 돌리는 루프를 짜서 금방 찾아내실 수 있을 겁니다.

혹시 지금 숫자 8자리로 잠긴 파일을 풀려고 고민 중이신가요? 아니면 보안 시스템 설계 관점에서 고려 중이신가요?


----------------------------

결론부터 말씀드리면, **네, 파이썬으로 가능합니다.**

사용자 암호(열기 암호)를 알고 있어서 일단 문서를 열 수만 있다면, 설령 편집 암호(권한 암호)를 모르더라도 파이썬 라이브러리를 이용해 **암호가 전혀 없는 깨끗한 텍스트 PDF**로 다시 저장할 수 있습니다.

그 이유와 방법을 개발자 관점에서 명확히 짚어드릴게요.

---

### 1. 왜 파이썬으로는 가능한가?
사람이 사용하는 Adobe Acrobat 같은 전용 뷰어는 PDF 표준 규약을 엄격히 따릅니다. 편집 암호가 걸려 있으면 "소프트웨어 차원"에서 수정 버튼을 비활성화하거나 저장을 막아버리는 거죠.

하지만 파이썬 라이브러리(특히 `pikepdf`나 `PyMuPDF`)는 문서를 로드한 뒤, 내부 데이터를 추출해서 **아무런 제약 조건이 설정되지 않은 새로운 PDF 객체**에 복사해 넣을 수 있습니다. 즉, 잠긴 문을 열고 들어가서 알맹이만 꺼내 새 상자에 담는 것과 같습니다.

### 2. 추천 도구: `pikepdf`
이런 "암호 제거 및 재저장" 작업에 가장 강력한 라이브러리는 `pikepdf`입니다. QPDF라는 강력한 C++ 엔진을 기반으로 하기 때문에 복잡한 권한 설정을 무시하고 새 파일로 만드는 데 최적화되어 있습니다.

**구현 로직 (예시 코드):**
```python
import pikepdf

# 열기 암호(a)만 넣고 파일을 엽니다.
# 편집 암호를 몰라도 열기 암호만으로 데이터 스트림 복호화는 이미 끝납니다.
with pikepdf.open("locked.pdf", password="a") as pdf:
    # 아무런 제약 조건 없이 새 파일로 저장합니다.
    # 이 과정에서 편집 암호, 복사 제한, 인쇄 제한 등이 모두 증발합니다.
    pdf.save("unlocked_clean.pdf")
```

### 3. 기술적 핵심 포인트
* **복호화의 주체:** 열기 암호(`a`)를 입력하는 순간, PDF 내부의 실제 콘텐츠(텍스트, 이미지 등)는 이미 복호화되어 메모리에 올라옵니다. 
* **권한의 무력화:** 편집 암호는 "이 데이터를 수정해도 되는가?"라는 '플래그(Flag)' 값에 불과합니다. 파이썬 프로그램은 이 플래그를 무시하고 데이터를 읽어서 새 파일에 쓸 수 있습니다.
* **결과물:** 생성된 `unlocked_clean.pdf`는 암호가 아예 없는 일반 PDF가 되며, 텍스트 원본 형식도 그대로 유지됩니다.

---

### 4. 만약 파이썬으로도 안 된다면? (드문 경우)
최신 보안 표준 중 하나인 **"Metadata Encryption"**까지 아주 강력하게 걸려 있는 경우, 일부 구형 라이브러리는 헤맬 수 있습니다. 하지만 앞서 말씀드린 `pikepdf`나 최신 버전의 `PyMuPDF`를 사용하신다면 99% 확률로 해결됩니다.

### 요약
사람은 뷰어 프로그램의 **'UI 제약'** 때문에 편집 암호 없이는 아무것도 못 하지만, **파이썬은 데이터를 직접 제어**하기 때문에 열기 암호만 있다면 암호 없는 깔끔한 PDF로 복제해내는 것이 가능합니다.
